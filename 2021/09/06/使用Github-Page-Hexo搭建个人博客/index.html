使用Github Page+Hexo搭建个人博客

Posted on 2021-09-06 Edited on 2021-09-07

前言

什么是Github Page?

​ GitHub Pages是GitHub 官方提供的静态站点托管服务，可以直接从Github仓库获取HTML、CSS和JavaScript文件，通过build process将其发布为一个静态网站。

什么是Hexo?

​ Hexo 是一个快速、简洁且高效的博客框架。

准备工作

注册 Github并创建仓库

​ GitHub Pages是GitHub 官方提供的静态站点托管服务，所以首先要注册一个GitHub账号。接下来为站点创建一个仓库，需要注意的是仓库必须命名为**.github.io**，并且user必须为小写。

安装Git

安装Node.js

Hexo是基于Node.js的静态博客框架，因此安装Hexo之前必须先安装Node.js。

安装Hexo

如果以上环境准备好了就可以使用 npm 开始安装 Hexo 了。

安装 hexo-cli

🚩 执行npm install命令时报了一些莫名其妙的错误，可能是网络的原因，改为cnpm install即可成功执行。

更换主题

如果不喜欢Hexo的默认主题，可以更换其他主题或是创建Hexo主题。这里选择的是Next主题，当然也可以去Hexo主题列表选择自己喜欢的风格的主题。

验证主题

首先，使用hexo clean 命令清除Hexo的缓存。

然后，运行命令hexo s --debug，以调试模式启动 Hexo 本地站点。

当命令行输出中提示出：

使用浏览器访问 http://localhost:4000，检查站点是否正确运行。

🚩 第一次启用Next主题，使用浏览器访问时页面显示如下：

从网上搜索了一下，问题的原因是Hexo在5.0之后把swig删除了，swig文件缺失导致。解决办法就是手动安装swig npm install hexo-renderer-swig --save ，成功安装swig后，问题依旧，后来才发现是Next主题版本的问题，网上好多教程都是使用了一个博主的仓库，使用的不是最新版本。安装Next主题时需要注意下仓库的地址，通过命令 $ git clone https://github.com/next-theme/hexo-theme-next themes/next 安装最新的官方版本即可解决上述问题。
